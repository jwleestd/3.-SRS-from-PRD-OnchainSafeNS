## Part B. 섹션별 결함 상세 및 수정 방향

### B-1. §1.4 해결 대상 문제

**결함:** "실패 KPI (현 상태)" 컬럼만 있고 "목표 KPI", "측정 경로" 컬럼이 없다. 현재 문제의 크기는 알 수 있으나, 제품이 이를 얼마나 개선해야 성공인지 판단 불가.

**수정 방향:**
- "목표 KPI (Phase별)" 컬럼 추가 — Phase 완료 시 도달해야 하는 수치 목표
- "측정 경로" 컬럼 추가 — 어떤 도구/방법으로 측정하는지 명시

---

### B-2. §5.1~5.4 기능 요구사항 AC

**결함 유형 3가지:**

| 결함 유형 | 해당 REQ 수 | 예시 |
|---|---|---|
| (a) GWT 미적용: 수치만 나열 | 19건(Phase-0 전체) + 후속 Phase | "≤ 2초(p95), 정확도 100%" |
| (b) AC 공란 ("—") | 12건 | FUNC-002~005, 021~027, 036, 042, 067~069 |
| (c) 실패/예외 케이스 누락 | 전 항목 | 잘못된 주소 형식, 미인증 접근, 네트워크 타임아웃 등 |

**수정 방향:**
- Phase-0 전 항목(19 REQ)을 Given-When-Then 구조로 재작성
- 각 REQ에 최소 1개 실패 시나리오 추가
- AC 공란 항목에 수치 기준 보충 (Phase-1~3은 대표 항목만 수정)

---

### B-3. §6 비기능 요구사항

**결함:**
- Phase-0 NFR: 임계치는 있으나 "위반 시 알림 경로", "모니터링 도구", "에스컬레이션 절차" 미정의
- REQ-P0-NF-009 "100+20+500건" — 500건의 대상 엔터티 불명확
- REQ-P0-NF-010 "Vercel Logs 추적 가능" — 추적 대상과 보존 기간 불명확
- Phase-1~3 추가 NFR: 번호 범위 묶음(043~044, 045~047 등)이 개별 기준 미분리

**수정 방향:**
- Phase-0 NFR에 "모니터링 도구" + "위반 시 조치" 컬럼 추가
- 불명확 항목 구체화

---

### B-4. §4 Differential Value (신규 섹션 필요)

**결함:** 기존 솔루션(GoPlus, ScamSniffer, Chainalysis, MetaMask 내장 경고) 대비 본 플랫폼의 차별 가치가 수치로 제시되지 않음. "왜 이 제품인가?"에 대한 정량적 답변 부재.

**수정 방향:**
- §4에 "4.4 Differential Value — 대안 대비 차별 가치" 하위 섹션 신설
- 기능 커버리지, 비용, 정확도, 사용자 경험 축에서 비교 테이블 제시

---

### B-5. §13 검증 계획

**결함:**
- Phase-0 (4건): KPI·성공 기준은 있으나 **기준선(Baseline)·측정 방법·샘플 크기·측정 도구** 부재
  - H-P0-1 "중단율 ≥80%" — 누가, 언제, 어떻게 측정?
  - H-P0-2 "사용률 ≥50%" — 분모 정의 없음 (전체 등록 유저? DAU?)
  - H-P0-4 "이해도 ≥ 90%" — 설문? 관찰? 객관식?
- Phase-1~3 (19건): **KPI·성공 기준 컬럼 자체 없음**. 가설과 Phase만 나열

**수정 방향:**
- Phase-0: "기준선", "측정 방법", "샘플/기간" 컬럼 추가
- Phase-1~3: 전 항목에 KPI·성공 기준·측정 방법 보충

### [수정] §4.4 Differential Value — 대안 대비 차별 가치 *(신규 섹션)*

> 본 섹션은 기존 시장 솔루션 대비 본 플랫폼의 차별적 가치를 정량 비교한다. 비교 대상은 Phase-0 시점에서 유사 기능을 제공하는 무상/유상 도구이다.

| 비교 축 | GoPlus Security API | ScamSniffer (브라우저 확장) | Chainalysis KYT | **본 플랫폼 (Phase-0)** | **본 플랫폼 (North Star)** |
|---|---|---|---|---|---|
| **사기 주소 조회** | ✅ API 제공 | ✅ 브라우저 탐지 | ✅ 엔터프라이즈 API | ✅ 웹 UI + API | ✅ Zero-FP API |
| **사용자 신고 채널** | ❌ 없음 | ❌ 없음 | ❌ 없음 | **✅ 웹 신고 + 운영 승인** | ✅ + Fraud Agent 자동 수집 |
| **Human-Readable 이름** | ❌ | ❌ | ❌ | **✅ Safe-Name (.safe)** | ✅ + NRM (ENS, .crypto 등) |
| **이체 전 사기 검증** | ❌ (조회만) | ❌ (경고만) | △ (VASP 전용) | **✅ 데모 이체 시뮬레이션** | ✅ Pre-Transfer + Hard Block |
| **오송금 보상** | ❌ | ❌ | ❌ | ❌ (Phase-3) | ✅ Warranty $30K |
| **월 비용 (도입 기관)** | 무상 (제한적) | 무상 | $5,000~$50,000+/월 | **$0~$21** | $500~$1,000 |
| **B2C 직접 접근** | ❌ (API만) | ✅ (Chrome 한정) | ❌ (B2B만) | **✅ 웹 직접 접근** | ✅ |
| **오탐지율 (FP)** | 비공개 | 비공개 | ≤ 0.1% (공개 자료) | 시드 기반 0% (한정) | **≤ 0.01% 목표** |

**핵심 차별 포인트:** (1) 신고→승인→DB 반영 폐루프, (2) Safe-Name 이름 기반 이체, (3) 월 $21 이하 MVP 운영, (4) B2C 직접 접근 가능. 이 4가지는 기존 솔루션에서 동시에 제공하지 않는 조합이다.
