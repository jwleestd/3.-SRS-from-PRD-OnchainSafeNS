## Part B. 섹션별 결함 상세 및 수정 방향

### B-1. §5.3 FUNC-053~061 번들 해체 (신규 — Medium)

**결함:** 9개 REQ(KYC Tier 4단계, Pre-Transfer, Compatibility Gate, Verified 배지)가 단일 행으로 묶여 AC가 "정확도 100%" 하나뿐이다. 개별 REQ의 입력·출력·실패 케이스가 구분되지 않아 테스트 케이스를 도출할 수 없다.

**수정 방향:** 핵심 Must 항목 3건(KYC 등록 → Pre-Transfer 검증 → Compatibility Gate)을 대표로 분리하고 개별 AC를 부여한다.

---

### B-2. §6.2 메인 NFR 테이블 포맷 불일치 (신규 — Medium)

**결함:** Phase-0 NFR(§6.1)은 6컬럼(ID·카테고리·요구사항·기준·모니터링 도구·위반 시 조치)이지만, Phase-1~3 메인 NFR(§6.2 성능/신뢰성/보안/비용)은 4컬럼(ID·요구사항·기준·Phase)이다. "추가 NFR" 하위 테이블은 5컬럼(+측정 방법)이다. 동일 문서 내 3가지 포맷이 공존하여 NF-001~030의 모니터링·위반 조치가 정의되지 않은 상태이다.

**수정 방향:** §6.2 메인 테이블 4개에 "측정 방법"과 "위반 시 조치" 컬럼을 추가하여 Phase-0 NFR과 포맷을 통일한다.

---

### B-3. §14 Phase 전환 기준 정량화 (신규 — Low)

**결함:** 일부 전환 조건이 정성적이다.
- §14.1 #4 "유저 피드백 10건+" — "피드백"의 정의(인터뷰? 설문? NPS?)와 충족 판정 기준이 없음
- §14.2 #2 "VASP 1사 핫라인 시범" — 파일럿 성공/실패 판정 기준이 없음
- §14.3 #4 "보험사 MOU" — MOU 체결만으로 충분한지, 구속력 있는 조항 필요 여부 불명확

**수정 방향:** 각 조건에 판정 기준(Pass/Fail Criteria) 컬럼을 추가한다.

---

### B-4. §15 Simulation 체크리스트 Pass/Fail (신규 — Low)

**결함:** 10개 항목 각각에 "이것이 통과하려면 무엇이 확인되어야 하는가"의 구체 기준이 없다. 예: "①외부 API 키 확인 — 환경변수+테스트"는 "어떤 API의 어떤 호출이 HTTP 200을 반환하면 Pass인가?"가 불명확하다.

**수정 방향:** 각 항목에 "Pass 기준" 컬럼을 추가한다.

---

### B-5. §4.4 Differential Value 벤치마크 방법론 (잔존 — Low)

**결함:** v0.3 리뷰에서 잔존 리스크로 식별됨. 비교 테이블의 "비공개" 항목에 대해 검증 방법이 정의되지 않았다.

**수정 방향:** 비교 테이블 하단에 "벤치마크 검증 계획" 단락을 추가한다.
