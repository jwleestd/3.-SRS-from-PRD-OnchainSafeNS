# 변경 사항 보고서: SRS v0.5 → PRD v0.3

**작성일:** 2026-05-14  
**입력 문서:** `0_SRS_v1_opus46_Fn.md` (SRS-001 v0.5, 2026-05-09)  
**출력 문서:** `3.PRD_v0.3.md` (PRD-003 v0.3, 2026-05-14)

---

## 1. 변환 목적

SRS(소프트웨어 요구사항 명세서)는 "시스템이 무엇을 해야 하는가"에 초점을 둔 기술 문서이며, PRD(제품 요구사항 문서)는 "왜 만드는가, 누구를 위한 것인가, 무엇을 만드는가"를 제품 관점에서 정리하는 문서이다. 본 변환은 SRS v0.5의 모든 지식을 유실 없이 보존하면서 PRD 체계로 재구조화하는 것을 목표로 했다.

## 4. 구조 변경 설계 의도

| # | 변경 | 설계 의도 |
|---|---|---|
| 1 | 제품 개요를 §1로 선행 | PRD 독자(PM, 경영진, 투자자)가 "왜 만드는가"를 첫 페이지에서 파악 |
| 2 | Phase 로드맵을 §2로 승격 | 점진적 구현 전략이 본 제품의 핵심 의사결정이므로 초반 배치 |
| 3 | 사용자 플로우를 독립 섹션으로 분리 | 기술 시스템 컨텍스트(SRS §3)와 사용자 경험 관점(PRD)을 구분 |
| 4 | 기술 아키텍처 통합 | SRS에 분산된 기술 정보(제약·외부 시스템·API·프로젝트 구조)를 하나로 모아 개발팀 참조 효율 향상 |
| 5 | Appendix 항목 독립 승격 | SRS Appendix에 묻혀 있던 비용·검증·시드 데이터를 독립 섹션으로 올려 의사결정 가시성 확보 |
| 6 | 용어·참조를 문서 말미로 이동 | PRD 본문 흐름을 "Why → Who → What → How → Verify"로 정리 |
