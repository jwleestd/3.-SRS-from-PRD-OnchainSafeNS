## 1. 기술 스택의 명확성

### 1.1 DB 환경 전략

| 항목 | v0.6 | v0.8 | 변경 사유 |
|---|---|---|---|
| C-TEC-003 DB 구성 | Prisma + SQLite(로컬) / Supabase(배포) | Prisma + **Supabase PostgreSQL 단일** (로컬·배포 모두 원격 DB) | Prisma의 `provider`와 URL 스킴 불일치 시 에러 발생하는 기술적 오류 해소. 초급 개발자의 환경 분기 설정 실패 원천 제거 |
| C-TEC-010 | SQLite→PostgreSQL 마이그레이션 (Phase-1~) | **(삭제)** — C-TEC-003으로 통합 | 이중 환경 자체가 제거되어 마이그레이션 불필요 |
| Prisma 호환 규칙 | SQLite/PostgreSQL 양쪽 호환 테이블 (JSON→String+parse, Enum→String 등) | PostgreSQL 단일 테이블 (JSON→네이티브 `Json`, Enum→네이티브 `enum`) | 단일 DB이므로 호환 규칙 자체가 불필요. PostgreSQL 네이티브 타입 직접 사용 가능 |
| datasource 설정 | `provider = "postgresql"` + "URL 스킴으로 SQLite 자동 감지" (기술적 오류) | `provider = "postgresql"` + `url = env("DATABASE_URL")` (Supabase 연결 문자열) | v0.6의 자동 감지 설명은 Prisma 실동작과 불일치하는 오류였음을 명시 |

### 1.2 아키텍처 계층 구조

| 항목 | v0.6 | v0.8 | 변경 사유 |
|---|---|---|---|
| 아키텍처 명칭 | Class Diagram (§3.5) | **Module Architecture Diagram** (§3.5) | Class Diagram은 OOP 설계 문서의 인상. 실제 Next.js 구조(파일 기반 라우팅)에 맞는 모듈 계층으로 전환 |
| 계층 수 | **(미명시)** — Service Module이 존재하나 계층 규칙 없음 | **2계층 명시**: Route Handler → Prisma 직접 호출 허용. 공통 로직만 `lib/` 분리 | 바이브코딩에서 AI가 생성하는 코드 구조(route.ts 단일 파일)와 일치시킴 |
| Service Module | `fraud.ts`, `safe-name.ts`, `auth.ts` (3개 필수) | **(Phase-0 필수 아님)** — Route Handler 내에서 직접 구현 허용. Phase-1a에서 100줄 초과·3회 반복 시 추출 | Phase-0 파일 수 감소, AI 코드 생성 결과와 구조 불일치 해소 |
| Component Diagram | `DB[("Supabase PostgreSQL")]` 단일 표기 | Supabase PostgreSQL 단일 + `lib/` 공유 모듈(email.ts, validators.ts, prisma.ts) 명시 | 어떤 로직이 공유되고 어떤 로직이 route.ts에 인라인되는지 구분 |

## 2. MVP 목표 및 가치전달 조정 내용

### 2.1 Phase 구조 재설계

| 항목 | v0.6 | v0.8 | 변경 사유 |
|---|---|---|---|
| Phase 수 | 4단계 (Phase-0, 1, 2, 3) | **5단계** (Phase-0, **1a**, **1b**, 2, 3) | Phase-0→1 난이도 절벽 해소. Phase-1의 20 FUNC + 8 외부연동을 CRUD(1a)와 외부연동(1b)으로 분할 |
| Phase-1a 범위 | — | 이의 신청, DNS 관리, 알림(Email+Slack 2채널), Fraud Agent 대시보드(수동). **블록체인·외부 API 없음** | 바이브코딩으로 구현 가능한 CRUD 확장만 포함 |
| Phase-1b 범위 | — | 핫라인+PagerDuty, 외부 소스 수집(2~3종), KakaoTalk(사업자등록 전제), SMS | 외부 API 연동 + 사업 요건(사업자등록)이 전제되는 기능 분리 |
| Phase-1 기간 | 4~6주 (단일) | **3~4주 + 3~4주** = 6~8주 (분할) | 총 기간 유사하나, 중간 Gate(§7.2)에서 품질 확인 가능 |
