### MW-001 — middleware.ts Admin 인증 가드

```typescript
// middleware.ts (프로젝트 루트)
import { NextResponse } from 'next/server';
import type { NextRequest } from 'next/server';
import { createHash, timingSafeEqual } from 'node:crypto';

/**
 * 타이밍 공격 방어용 안전 비교
 * 단순 === 비교는 타이밍 사이드채널 공격에 취약하므로 timingSafeEqual 사용
 */
function safeCompare(a: string, b: string): boolean {
  if (!a || !b) return false;
  const aHash = createHash('sha256').update(a).digest();
  const bHash = createHash('sha256').update(b).digest();
  // sha256 결과는 항상 32바이트 → length 동일 보장
  return timingSafeEqual(aHash, bHash);
}

export function middleware(req: NextRequest) {
  const { pathname } = req.nextUrl;
  const adminSession = req.cookies.get('admin_session')?.value ?? '';
  const adminPassword = process.env.ADMIN_PASSWORD ?? '';
  const isAuthenticated = safeCompare(adminSession, adminPassword);

  // Admin 페이지 보호: /(admin)/* → /auth/admin-login 리다이렉트
  if (pathname.startsWith('/admin')) {
    if (!isAuthenticated) {
      const loginUrl = new URL('/auth/admin-login', req.url);
      loginUrl.searchParams.set('redirect', pathname);
      return NextResponse.redirect(loginUrl);
    }
  }

  // Admin API 보호: /api/v1/admin/* → 401 JSON
  if (pathname.startsWith('/api/v1/admin')) {
    if (!isAuthenticated) {
      return NextResponse.json(
        {
          success: false,
          error: { code: 'UNAUTHORIZED', message: '인증이 필요합니다' },
          meta: { timestamp: new Date().toISOString(), request_id: crypto.randomUUID() }
        },
        { status: 401 }
      );
    }
  }

  return NextResponse.next();
}

export const config = {
  matcher: ['/admin/:path*', '/api/v1/admin/:path*']
};
```

**🔐 보안 권장사항:**

| Phase | 권장 패턴 | 보안 강도 |
|---|---|---|
| **Phase-0 (현재)** | admin_session = ADMIN_PASSWORD 평문 + sha256 timing-safe 비교 | ⚠️ 쿠키 탈취 시 비밀번호 노출 (단, timing 공격은 방어) |
| **Phase-1a 권장** | admin_session = JWT (HS256, exp 1h) 또는 random session token + Redis 저장 | ✅ 토큰 분리, 자동 만료 |
| **Phase-2 권장** | OAuth + RBAC (CMD-CONSOLE-001) | ✅ 세분화된 권한 |

**Phase-0 운영 시 주의:**
1. **HTTPS 강제 의무** (NFR-002) — HTTP 쿠키 탈취 방어
2. **SameSite=Strict 권장** — CSRF 방어 (현재 'lax')
3. **ADMIN_PASSWORD 강도** — 최소 24자 랜덤 (예: `openssl rand -base64 24`)
4. **운영 메모:** Phase-0 출시 후 30일 내 Phase-1a JWT 도입 우선

### CMD-AUTH-001 — 이메일 인증 코드 발송

```typescript
// app/api/v1/auth/verify-email/route.ts (POST handler — send 분기)
import { NextRequest, NextResponse } from 'next/server';
import { randomInt } from 'node:crypto';  // ← 암호학적으로 안전한 난수 생성
import { prisma } from '@/lib/prisma';
import { sendVerificationCode } from '@/lib/email';
import { validateEmail } from '@/lib/validators';

export async function POST(req: NextRequest) {
  const body = await req.json();
  const { email, action } = body;

  // 1. 이메일 형식 검증
  if (!validateEmail(email)) {
    return NextResponse.json(
      { success: false, error: { code: 'VALIDATION_ERROR', message: '올바른 이메일 형식을 입력하세요' } },
      { status: 400 }
    );
  }

  if (action === 'send') {
    // 2. 1분 이내 재요청 차단
    const existing = await prisma.user.findUnique({ where: { email } });
    if (existing?.verification_expires_at) {
      const remainingMs = existing.verification_expires_at.getTime() - Date.now();
      if (remainingMs > 9 * 60 * 1000) {  // 9분+ 남음 = 1분 미만 경과
        return NextResponse.json(
          { success: false, error: { code: 'RATE_LIMITED', message: '잠시 후 다시 시도하세요' } },
          { status: 429 }
        );
      }
    }

    // 3. 6자리 코드 생성 (암호학적으로 안전) + DB 저장 + Resend 발송
    //    crypto.randomInt(100000, 1000000) → 100000 ≤ x < 1000000 (정확히 6자리)
    //    Math.random()은 예측 가능하므로 인증 코드에 사용 금지 (보안 권장사항)
    const code = String(randomInt(100000, 1000000));
    const expiresAt = new Date(Date.now() + 10 * 60 * 1000);  // 10분 후

    await prisma.user.upsert({
      where: { email },
      update: { verification_code: code, verification_expires_at: expiresAt },
      create: { email, verification_code: code, verification_expires_at: expiresAt, email_verified: false }
    });

    await sendVerificationCode(email, code);  // 비차단 처리

    return NextResponse.json(
      { success: true, data: { message: '코드가 발송되었습니다', expires_in_sec: 600 } },
      { status: 200 }
    );
  }
  // action === 'verify' 분기는 CMD-AUTH-002에서 처리
}
```

### CMD-AUTH-002 — 인증 코드 확인

```typescript
// app/api/v1/auth/verify-email/route.ts (POST handler — verify 분기)
// 위 CMD-AUTH-001 같은 route 내 action='verify' 분기

if (action === 'verify') {
  const { email, code } = body;

  // 1. 코드 형식 검증
  if (!/^\d{6}$/.test(code)) {
    return NextResponse.json(
      { success: false, error: { code: 'VALIDATION_ERROR', message: '6자리 코드를 입력하세요' } },
      { status: 400 }
    );
  }

  // 2. USER 조회
  const user = await prisma.user.findUnique({ where: { email } });
  if (!user || !user.verification_code) {
    return NextResponse.json(
      { success: false, error: { code: 'INVALID_CODE', message: '인증 코드를 먼저 발송하세요' } },
      { status: 401 }
    );
  }

  // 3. 코드 만료 확인
  if (user.verification_expires_at && user.verification_expires_at < new Date()) {
    return NextResponse.json(
      { success: false, error: { code: 'EXPIRED', message: '코드가 만료되었습니다' } },
      { status: 410 }
    );
  }

  // 4. 코드 일치 확인
  if (user.verification_code !== code) {
    return NextResponse.json(
      { success: false, error: { code: 'INVALID_CODE', message: '코드가 일치하지 않습니다' } },
      { status: 401 }
    );
  }

  // 5. email_verified=true + 코드 무효화
  await prisma.user.update({
    where: { user_id: user.user_id },
    data: {
      email_verified: true,
      verification_code: null,
      verification_expires_at: null
    }
  });

  // 6. 쿠키 설정 + 응답
  const response = NextResponse.json(
    { success: true, data: { user_id: user.user_id, email_verified: true } },
    { status: 200 }
  );
  response.cookies.set('user_session', user.user_id, {
    httpOnly: true,
    secure: true,
    sameSite: 'lax',
    maxAge: 7 * 24 * 60 * 60  // 7일
  });
  return response;
}
```

```typescript
// app/auth/verify/page.tsx
'use client';
import { useState } from 'react';
import { Input } from '@/components/ui/input';
import { Button } from '@/components/ui/button';
import { InputOTP, InputOTPGroup, InputOTPSlot } from '@/components/ui/input-otp';
import { toast } from 'sonner';

export default function VerifyEmailPage() {
  const [email, setEmail] = useState('');
  const [code, setCode] = useState('');
  const [step, setStep] = useState<'email' | 'code'>('email');
  const [loading, setLoading] = useState(false);

  const handleSendCode = async () => {
    setLoading(true);
    const res = await fetch('/api/v1/auth/verify-email', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ email, action: 'send' })
    });
    const data = await res.json();
    if (data.success) {
      toast.success('인증 코드가 발송되었습니다');
      setStep('code');
    } else {
      toast.error(data.error.message);
    }
    setLoading(false);
  };

  const handleVerifyCode = async () => {
    setLoading(true);
    const res = await fetch('/api/v1/auth/verify-email', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ email, code, action: 'verify' })
    });
    const data = await res.json();
    if (data.success) {
      toast.success('인증이 완료되었습니다');
      window.location.href = '/';
    } else {
      toast.error(data.error.message);
    }
    setLoading(false);
  };

  return (
    <div className="max-w-md mx-auto p-6 space-y-4">
      <h1 className="text-2xl font-bold">이메일 인증</h1>
      {step === 'email' ? (
        <>
          <Input
            type="email"
            placeholder="user@example.com"
            value={email}
            onChange={(e) => setEmail(e.target.value)}
          />
          <Button onClick={handleSendCode} disabled={loading || !email} className="w-full">
            인증 코드 요청
          </Button>
        </>
      ) : (
        <>
          <p className="text-sm text-muted-foreground">{email}로 전송된 6자리 코드를 입력하세요</p>
          <InputOTP maxLength={6} value={code} onChange={setCode}>
            <InputOTPGroup>
              {[0, 1, 2, 3, 4, 5].map((i) => (
                <InputOTPSlot key={i} index={i} />
              ))}
            </InputOTPGroup>
          </InputOTP>
          <Button onClick={handleVerifyCode} disabled={loading || code.length !== 6} className="w-full">
            인증 확인
          </Button>
        </>
      )}
    </div>
  );
}
```

```typescript
import { prisma } from '@/lib/prisma';
import { validateAddress } from '@/lib/validators';
import { SUPPORTED_CHAINS } from '@/lib/constants';

export async function GET(req: NextRequest) {
  const { searchParams } = new URL(req.url);

  const chain = searchParams.get('chain');

  // 1. 파라미터 누락

      { success: false, error: { code: 'VALIDATION_ERROR', message: '주소와 체인을 모두 입력하세요' } },
      { status: 400 }
    );
  }

  // 2. 미지원 체인
  if (!SUPPORTED_CHAINS.includes(chain as any)) {
    return NextResponse.json(
      { success: false, error: { code: 'VALIDATION_ERROR', message: '지원하지 않는 체인입니다' } },
      { status: 400 }
    );
  }

  // 3. 주소 형식 검증
  if (!validateAddress(address, chain)) {
    return NextResponse.json(
      { success: false, error: { code: 'VALIDATION_ERROR', message: '올바른 주소 형식을 입력하세요' } },
      { status: 400 }
    );
  }

  // 4. FRAUD_ADDRESS 조회 (인덱스 활용)
  const fraud = await prisma.fraudAddress.findFirst({
    where: { address: address.toLowerCase(), chain }
  });

  if (!fraud) {
    return NextResponse.json({
      success: true,
      data: { found: false, message: '등록된 사기 이력이 없습니다' }
    });
  }

  return NextResponse.json({
    success: true,
    data: {
      found: true,
      risk_level: fraud.risk_level,
      report_count: fraud.report_count,
      source_type: fraud.source_type ?? '출처 미확인'  // QRY-FRAUD-003 폴백
    }
  });
}
```

```typescript
// app/api/v1/fraud/report/route.ts (POST)
import { NextRequest, NextResponse } from 'next/server';
import { prisma } from '@/lib/prisma';
import { validateAddress } from '@/lib/validators';
import { sendReportConfirmation } from '@/lib/email';

export async function POST(req: NextRequest) {
  // 1. 인증 확인
  const userId = req.cookies.get('user_session')?.value;
  if (!userId) {
    return NextResponse.json(
      { success: false, error: { code: 'UNAUTHORIZED', message: '인증이 필요합니다' } },
      { status: 401 }
    );
  }

  const user = await prisma.user.findUnique({ where: { user_id: userId } });
  if (!user || !user.email_verified) {
    return NextResponse.json(
      { success: false, error: { code: 'UNAUTHORIZED', message: '이메일 인증이 필요합니다' } },
      { status: 401 }
    );
  }

  // 2. 제한 유저 확인 (CMD-FRAUD-004)
  if (user.report_restriction_until && user.report_restriction_until > new Date()) {
    return NextResponse.json(
      {
        success: false,
        error: {
          code: 'RATE_LIMITED',
          message: `신고 제한 중입니다. ${user.report_restriction_until.toISOString().split('T')[0]}까지`
        }
      },
      { status: 429 }
    );
  }

  // 3. 입력 검증
  const body = await req.json();
  const { reported_address, chain, description, evidence_url } = body;

  if (!validateAddress(reported_address, chain)) {
    return NextResponse.json(
      { success: false, error: { code: 'VALIDATION_ERROR', message: '올바른 주소 형식을 입력하세요' } },
      { status: 400 }
    );
  }
  if (!description || description.length < 10 || description.length > 1000) {
    return NextResponse.json(
      { success: false, error: { code: 'VALIDATION_ERROR', message: '피해 내역은 10~1000자' } },
      { status: 400 }
    );
  }

  // 4. 중복 신고 확인 (CMD-FRAUD-002)
  const existing = await prisma.fraudReport.findFirst({
    where: { reporter_id: userId, reported_address: reported_address.toLowerCase() }
  });
  if (existing) {
    return NextResponse.json(
      { success: false, error: { code: 'CONFLICT', message: '이미 신고한 주소입니다' } },
      { status: 409 }
    );
  }

  // 5. FRAUD_REPORT 생성 (Phase-0 정책: 신고는 'submitted' 상태로만 저장)
  //    FRAUD_ADDRESS 반영은 Admin 승인 시(CMD-ADMIN-001)에만 수행 — SRS REQ-P0-012 준수
  const report = await prisma.fraudReport.create({
    data: {
      reporter_id: userId,
      reported_address: reported_address.toLowerCase().trim(),
      chain,
      description,
      evidence_url: evidence_url || null,
      status: 'submitted'
    }
  });

  // 6. 이메일 발송 (비차단 — CMD-FRAUD-003)
  sendReportConfirmation({
    email: user.email,
    report_id: report.report_id,
    reported_address,
    chain,
    reported_at: report.reported_at
  }).catch((err) => console.error('[Email] sendReportConfirmation failed:', err));

  return NextResponse.json(
    { success: true, data: { report_id: report.report_id, status: 'submitted' } },
    { status: 201 }
  );
}
```

```typescript
  const baseName = human_name.toLowerCase().replace(/\.safe$/, '');

  // 3. 이름 규칙 검증
  if (!validateNameFormat(baseName)) {
    return NextResponse.json(
      { success: false, error: { code: 'VALIDATION_ERROR', message: '영소문자·숫자·하이픈, 3~20자' } },
      { status: 400 }
    );
  }

  // 4. 예약어 검증
  if (isReservedWord(baseName)) {
    return NextResponse.json(
      { success: false, error: { code: 'VALIDATION_ERROR', message: '예약된 이름입니다' } },
      { status: 400 }
    );
  }

  // 5. .safe 접미사 자동 부착
  const finalName = baseName + SAFE_NAME_SUFFIX;

  // 6. 중복 검증
  const existing = await prisma.safeName.findUnique({ where: { human_name: finalName } });
  if (existing) {
    return NextResponse.json(
      { success: false, error: { code: 'CONFLICT', message: '이미 사용 중인 이름입니다' } },
      { status: 409 }
    );
  }

  // 7. 주소 형식 검증 (CMD-SN-002)
  if (!validateAddress(wallet_address, chain)) {
    return NextResponse.json(
      {
        success: false,
        error: { code: 'VALIDATION_ERROR', message: `${chain}에 맞는 주소 형식이 아닙니다` }
      },
      { status: 400 }
    );
  }

  // 8. SAFE_NAME 생성
  const safeName = await prisma.safeName.create({
    data: {
      human_name: finalName,
      wallet_address: wallet_address.toLowerCase(),
      chain,
      owner_id: userId,
      status: 'active'
    }
  });

  return NextResponse.json(
    {
      success: true,
      data: { name_id: safeName.name_id, human_name: safeName.human_name, status: 'active' }
    },
    { status: 201 }
  );
}
```

### CMD-SN-002 — 지갑 주소 형식 검증

**📌 구현 위치:** 본 태스크의 검증 로직은 [CMD-SN-001 route handler](#cmd-sn-001--safe-name-등록-이름-규칙중복예약어) 내 7번 블록에 통합되어 있습니다. 핵심 검증 함수는 [LIB-003 validators.ts](#lib-003)에 정의되어 있습니다.

**검증 함수 (LIB-003 재게재):**

```typescript
// lib/validators.ts (LIB-003)
import { ADDRESS_PATTERNS } from './constants';

/**
 * 체인별 지갑 주소 형식 검증
 * @param address - 검증 대상 주소
 * @param chain - 체인 식별자 (ethereum, polygon, bsc, solana)
 * @returns 형식 일치 여부
 */
export function validateAddress(address: string, chain: string): boolean {
  if (!address || typeof address !== 'string') return false;
  if (!chain || typeof chain !== 'string') return false;

  const pattern = ADDRESS_PATTERNS[chain];
  if (!pattern) return false;

  // 공백 제거 후 검증
  return pattern.test(address.trim());
}
```

### CMD-ADMIN-001 — 신고 승인 + FRAUD_ADDRESS 반영

```typescript
// app/api/v1/admin/approve/route.ts (POST)
import { NextRequest, NextResponse } from 'next/server';
import { prisma } from '@/lib/prisma';
import { sendApprovalResult } from '@/lib/email';

export async function POST(req: NextRequest) {
  // MW-001 자동 인증 가드 (Admin 미인증 시 401)
  const body = await req.json();
  const { report_id, action, reviewer_notes } = body;

  const report = await prisma.fraudReport.findUnique({
    where: { report_id },
    include: { reporter: { select: { email: true } } }
  });

  if (!report) {
    return NextResponse.json(
      { success: false, error: { code: 'NOT_FOUND', message: '신고를 찾을 수 없습니다' } },
      { status: 404 }
    );
  }

  if (report.status !== 'submitted') {
    return NextResponse.json(
      { success: false, error: { code: 'VALIDATION_ERROR', message: '이미 처리된 신고입니다' } },
      { status: 400 }
    );
  }

  if (action === 'approve') {
    // Prisma transaction으로 원자성 보장
    const result = await prisma.$transaction(async (tx) => {
      // 1. FRAUD_REPORT 상태 갱신
      await tx.fraudReport.update({
        where: { report_id },
        data: { status: 'verified', reviewed_at: new Date() }
      });

      // 2. FRAUD_ADDRESS 존재 여부 확인 후 신규 생성 또는 카운트 증가
      //    (Phase-0 schema에는 (address, chain) 복합 unique가 없으므로
      //     findFirst + create/update 패턴 사용)
      const existing = await tx.fraudAddress.findFirst({
        where: {
          address: report.reported_address,
          chain: report.chain
        }
      });

      let fraudAddress;
      if (existing) {
        // 기존 사기 주소 — report_count 증가 + last_verified_at 갱신
        fraudAddress = await tx.fraudAddress.update({
          where: { fraud_id: existing.fraud_id },
          data: {
            report_count: { increment: 1 }
          }
        });
      } else {
        // 신규 사기 주소 등록 (운영자 승인 기반)
        fraudAddress = await tx.fraudAddress.create({
          data: {
            address: report.reported_address,
            chain: report.chain,
            risk_level: 'medium',          // 커뮤니티 신고는 medium 기본
            report_count: 1,
            source_type: 'community',
            first_reported_at: report.reported_at,
            status: 'verified'
          }
        });
      }

      return fraudAddress;
    });

    // 3. 이메일 통지 (CMD-ADMIN-003) — 비차단
    sendApprovalResult({
      email: report.reporter.email,
      report_id: report.report_id,
      action: 'verified',
      reviewer_notes
    }).catch((err) => console.error('[Email] sendApprovalResult failed:', err));

    return NextResponse.json({
      success: true,
      data: { status: 'verified', fraud_address_id: result.fraud_id }
    });
  }

  // action === 'reject' 분기는 CMD-ADMIN-002 참조
}
```

**참고:** Phase-1+ 에서는 `FRAUD_ADDRESS`에 `@@unique([address, chain])` 복합 인덱스를 추가하여 `upsert` 패턴 사용을 권장합니다 (DB-016 확장 시). Phase-0 에서는 위 `findFirst + create/update` 패턴이 SRS §6.2 정의에 맞습니다.

```typescript
const resend = new Resend(process.env.RESEND_API_KEY);

export async function sendApprovalResult(params: {
  email: string;
  report_id: string;
  action: 'verified' | 'rejected';
  reviewer_notes?: string;
}): Promise<boolean> {
  const subject = params.action === 'verified'
    ? '[Fraud Shield] 신고 승인'
    : '[Fraud Shield] 신고 거부';

  const body = params.action === 'verified'
    ? `<p>귀하의 신고(Report ID: <code>${params.report_id}</code>)가 승인되어 사기 DB에 반영되었습니다.</p>`
```

```typescript
export async function POST(req: NextRequest) {
  // 1. 인증 확인
  const userId = req.cookies.get('user_session')?.value;
  if (!userId) {
    return NextResponse.json(
      { success: false, error: { code: 'UNAUTHORIZED', message: '인증이 필요합니다' } },
      { status: 401 }
    );
  }

  const user = await prisma.user.findUnique({ where: { user_id: userId } });
  if (!user?.email_verified) {
    return NextResponse.json(
      { success: false, error: { code: 'UNAUTHORIZED' } },
      { status: 401 }
    );
  }

  const body = await req.json();
  const { recipient_name, amount } = body;

  // 2. amount 검증
  if (typeof amount !== 'number' || amount <= 0) {
    return NextResponse.json(
      { success: false, error: { code: 'VALIDATION_ERROR', message: '유효한 송금액을 입력하세요' } },
      { status: 400 }
    );
  }

  // 3. SAFE_NAME 리졸브
  const finalName = recipient_name.toLowerCase().endsWith(SAFE_NAME_SUFFIX)
    ? recipient_name.toLowerCase()
    : recipient_name.toLowerCase() + SAFE_NAME_SUFFIX;

  const safeName = await prisma.safeName.findUnique({ where: { human_name: finalName } });

  // 4. 미등록 → name_not_found
  if (!safeName || safeName.status !== 'active') {
    const transfer = await prisma.transferDemo.create({

        sender_id: userId,
        recipient_name: finalName,
        resolved_address: null,
        chain: null,

      }
    });
```

## 부록 C — v0.2 → v0.4 변경 이력 (품질 결함 보강)

### v0.2 → v0.4 변경 요약

본 v0.4는 v0.2(3-2.ISSUES_Com_V0_2_opus47.md) 검토 결과 식별된 **13가지 품질 결함**을 모두 보강한 최종본입니다.

#### 🔴 P0 치명적 결함 (3건) — 동작 불가능 또는 SRS 위반

**1. CMD-ADMIN-001 — `__placeholder__` 동작 불가 코드**
- **결함:** Prisma upsert의 where 절에 `fraud_id: '__placeholder__'` 사용. fraud_id는 PK이므로 신규 생성 시 항상 미존재 → upsert 자체가 부적절. 빌드는 통과하나 런타임에 매번 신규 생성 또는 실패.
- **수정:** `prisma.$transaction` + `findFirst` + 조건 분기(`create` 또는 `update`) 패턴으로 교체. 트랜잭션으로 원자성 보장. Phase-1+ `@@unique([address, chain])` 추가 권장 메모 포함.

**2. CMD-ADMIN-003 — 존재하지 않는 `EmailRetryQueue` 모델**
- **결함:** `prisma.emailRetryQueue.create(...)` 호출. SRS §6.2 Phase-0 6개 모델(USER, FRAUD_ADDRESS, FRAUD_REPORT, SAFE_NAME, OPERATOR, TRANSFER_DEMO)에 해당 모델 없음 → 컴파일 에러.
- **수정:** 큐 저장 코드 제거 + `console.error` 구조화 로그로 대체. 재시도 정책을 Phase-0(로그만)과 Phase-1+(Vercel Cron + 신규 테이블)로 분리 명시.

**3. CMD-FRAUD-001 — SRS REQ-P0-012 위반**
- **결함:** 사기 신고 접수 시 즉시 `prisma.fraudAddress.updateMany({ data: { report_count: { increment: 1 }}})` 실행. SRS 정책은 "Admin 승인 시에만 FRAUD_ADDRESS 갱신"인데 무단 갱신 발생.
- **수정:** updateMany 블록 완전 제거. `reported_address.toLowerCase().trim()` 추가로 sanitization 강화. SRS 정책 주석 추가.

#### 🔴 P1 보안 결함 (2건) — 프로덕션 보안 약점

**4. CMD-AUTH-001 — `Math.random()` 보안 취약**
- **결함:** `Math.floor(100000 + Math.random() * 900000)`으로 인증 코드 생성. Math.random은 PRNG 시드가 예측 가능 → 공격자가 코드 추측 가능.
- **수정:** `node:crypto`의 `randomInt(100000, 1000000)` 사용. 6자리 범위 정확 보장 + 암호학적으로 안전. 보안 권장사항 주석 추가.

**5. MW-001 — 타이밍 공격 + 쿠키 평문 비교**
- **결함:** `adminSession === process.env.ADMIN_PASSWORD` 단순 비교. 타이밍 사이드채널 공격에 취약 + 쿠키가 비밀번호 평문 그대로.
- **수정:** `createHash('sha256')` 해싱 후 `timingSafeEqual` 비교로 변경. Phase-0 한계(쿠키 = 비밀번호 평문) 명시 + Phase-1a JWT 도입 권장 메모 + ADMIN_PASSWORD 강도 가이드.
